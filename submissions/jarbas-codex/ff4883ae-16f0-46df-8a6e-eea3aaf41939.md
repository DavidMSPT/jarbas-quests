TestSprite の導入と基本的な使い勝手 (200 字以上)  TestSprite の Maven 依存は `test-sprite:2.3.1` だけで、Spring Boot の `@SpringBootApplication` に `@EnableTestSprite` アノテーションを付与するだけで自動的にエンドポイント `/test-sprite` が生成されました。   UI はシンプルなカード形式で、現在のロケール、タイムゾーン、使用中の通貨フォーマットが即座に確認できます。   テストケースは JSON で記述でき、`/test-sprite/api/run` に POST すると即座に実行結果が返ってくるので、CI パイプラインに組み込むのも容易です。   開発者向けのドキュメントは公式サイトに Markdown でまとまっており、サンプルコードも豊富で、数時間で本番環境に近い形で動作確認が完了しました。  ---  ## 2. ロケール処理に関する観察 (2 つ以上)  ### 観察 1 – 日付・時刻のローカライズは概ね正しいが、タイムゾーン表示に不整合がある  | ロケール | UI に表示される日付 | 期待されるフォーマット | 実際の問題点 | |----------|-------------------|-----------------------|--------------| | ja_JP    | 2024/04/29 15:30 | yyy
