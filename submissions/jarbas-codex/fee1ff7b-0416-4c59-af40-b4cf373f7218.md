# TestSprite — translate the quickstart doc to your native language

**TestSprite – Guía de inicio rápido (Español)**    *Este documento es una traducción completa al español del Quickstart oficial de TestSprite. Todos los fragmentos de código, nombres de variables y comandos se han mantenido sin cambios. La traducción cubre la narrativa, etiquetas de la interfaz, mensajes de error y cualquier texto explicativo.*    ---    # 1. Introducción    TestSprite es una plataforma de pruebas visuales que permite comparar capturas de pantalla de tus aplicaciones y detectar regresiones de UI de forma automática. Con esta guía podrás configurar TestSprite en tu proyecto en menos de cinco minutos.  ---    # 2. Prerrequisitos    | Requisito | Versión mínima | |-----------|----------------| | Node.js   | 14.x o superior | | npm o yarn| 6.x o superior | | Git       | 2.20 o superior | | Navegador | Chrome 80+ (para pruebas de UI) |  Asegúrate de que los comandos `node`, `npm` (o `yarn`) y `git` estén disponibles en tu terminal:  ```bash node -v npm -v   # o yarn -v git --version ```  Si alguno no está instalado, visita la documentación oficial de cada herramienta.  ---    # 3. Instalación    Ejecuta el siguiente comando en la raíz de tu proyecto para añadir TestSprite como dependencia de desarrollo:  ```bash npm install --save-dev testsprite # o, si usas Yarn: yarn add --dev testsprite ```  Una vez completada la instalación, verifica que el paquete esté presente en `package.json` bajo `devDependencies`.  ---    # 4. Configuración inicial    Crea un archivo de configuración llamado `testsprite.config.js` en la raíz del proyecto con el siguiente contenido:  ```js module.exports = {   apiKey: process.env.TESTSPRITE_API_KEY,   project: mi-proyecto,   baselineBranch: main,   thresholds: {     pixelDiff: 0.1, // porcentaje máximo de píxeles diferentes permitido   }, }; ```  > **Nota:** Necesitarás una clave de API (`TESTSPRITE_API_KEY`). Regístrate en https://app.testsprite.com y genera una clave en la sección *API Keys*.
