Sólo se traducen los textos narrativos, etiquetas de la interfaz y mensajes de error.  ## 1. Introducción    TestSprite es una herramienta ligera para crear pruebas unitarias visuales de componentes UI. Con ella puedes comparar capturas de pantalla de tus componentes entre diferentes versiones y detectar regresiones visuales de forma automática.  ## 2. Prerrequisitos    Antes de comenzar, asegúrate de tener instalados los siguientes elementos:  | Requisito | Versión mínima | Comentario | |-----------|----------------|------------| | **Node.js** | 14.x o superior | Descarga desde https://nodejs.org/ | | **npm** | 6.x o superior | Se instala junto con Node.js | | **Git** | 2.20 o superior | Necesario para clonar el repositorio |  ## 3. Instalación    Ejecuta el siguiente comando en la raíz de tu proyecto para añadir TestSprite como dependencia de desarrollo:  ```bash npm install --save-dev testsprite ```  ### 3.1 Verificación de la instalación    Para confirmar que la instalación fue exitosa, ejecuta:  ```bash npx testsprite --version ```  Deberías obtener una salida similar a:  ``` TestSprite CLI v1.4.2 ```  > **Error común:**   > `Error: comando testsprite no encontrado`   > *Solución:* Asegúrate de que la carpeta `node_modules/.bin` está en tu `PATH` o ejecuta el comando con `npx`.  ## 4.
