# TestSprite — translate the quickstart doc to your native language

**TestSprite – Guía de inicio rápido (Español)**    *Versión traducida para desarrolladores hispanohablantes*    ---    ## 1. Introducción    TestSprite es una plataforma de pruebas automatizadas que permite crear, ejecutar y gestionar pruebas de UI y API de forma sencilla. Esta guía de inicio rápido le mostrará cómo instalar el CLI, crear su primer proyecto de pruebas y ejecutar sus pruebas locales.    ---    ## 2. Prerrequisitos    - **Node.js** (versión ≥ 14) instalado y accesible desde la línea de comandos.   - Una cuenta en **TestSprite** (regístrese en https://app.testsprite.com).   - Acceso a Git (opcional, pero recomendado).    ---    ## 3. Instalación del CLI    ```bash npm install -g @testsprite/cli ```  > **Nota:** El nombre del paquete y los comandos deben mantenerse tal cual; **no** traducir `@testsprite/cli`.    Verifique la instalación:    ```bash testsprite --version ```  Salida esperada:    ``` testsprite/cli 1.2.3 ```    ---    ## 4. Autenticación    Ejecute el siguiente comando para iniciar sesión con sus credenciales de TestSprite:    ```bash testsprite login ```  Se abrirá una ventana del navegador con la URL de autenticación de TestSprite. Después de iniciar sesión, la CLI mostrará:    ``` ✅ Autenticado correctamente. ```    ---    ## 5. Creación de un proyecto de prueba    Cree una nueva carpeta para su proyecto y navegue a ella:    ```bash mkdir mi-primer-proyecto-testsprite cd mi-primer-proyecto-testsprite ```    Inicialice el proyecto con el asistente interactivo:    ```bash testsprite init ```    El asistente le pedirá que seleccione un **framework** (por ejemplo, `playwright`, `cypress` o `puppeteer`). Elija el que prefiera y responda a las preguntas restantes (nombre del proyecto, directorio de pruebas, etc.).
