Introducción    TestSprite es una plataforma de pruebas automatizadas que permite crear, ejecutar y analizar pruebas de UI de forma sencilla. Este documento rápido (quickstart) le guiará paso a paso para que pueda integrar TestSprite en su proyecto y comenzar a ejecutar sus primeras pruebas en menos de 10 minutos.  ---  #### 2. Requisitos previos    - **Node.js** ≥ 14.x (https://nodejs.org/)   - **npm** ≥ 6.x (se instala junto con Node)   - Una cuenta activa en **TestSprite** (regístrese en https://app.testsprite.com)   - Acceso a la terminal o línea de comandos de su sistema operativo    ---  #### 3. Instalación del cliente CLI    ```bash npm install -g testsprite-cli ```  Este comando instala globalmente la herramienta de línea de comandos `testsprite`. Después de la instalación, verifique que esté disponible:  ```bash testsprite --version ```  Debería mostrarse la versión actual, por ejemplo `testsprite-cli 1.3.2`.  ---  #### 4. Autenticación    Ejecute el siguiente comando para iniciar sesión con sus credenciales de TestSprite:  ```bash testsprite login ```  Se abrirá una ventana del navegador solicitándole su **correo electrónico** y **contraseña**. Tras autenticarse, la CLI guardará un token de acceso en `~/.testsprite/token.json`.  > **Mensaje de error típico:**   > `Error: Unable to authenticate – please check your credentials`   > *Traducción:* “Error: No se pudo autenticar – verifique sus credenciales”.  ---  #### 5.
