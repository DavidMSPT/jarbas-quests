Instalación    ```bash npm install -g testsprite-cli ```  > **Error típico:**   > `EACCES: permission denied, access /usr/local/lib/node_modules`   > Solución: ejecute el comando con `sudo` o configure npm para instalar paquetes globales en su directorio de usuario.  ## 2. Crear un proyecto    ```bash testsprite init mi‑proyecto cd mi‑proyecto ```  El comando `testsprite init` genera la siguiente estructura de carpetas:  ``` mi-proyecto/ ├─ src/ │  └─ index.js ├─ tests/ │  └─ index.test.js ├─ .testsprite/ │  └─ config.json └─ package.json ```  ## 3. Configuración básica    Edite ` .testsprite/config.json ` para definir el entorno de pruebas:  ```json {   browser: chrome,   headless: true,   baseUrl: http://localhost:3000 } ```  **Campos importantes:**    - **browser** – Navegador a usar (`chrome`, `firefox`, `edge`).   - **headless** – Ejecutar en modo sin cabeza (sin UI).   - **baseUrl** – URL base de la aplicación bajo prueba.  ## 4.
