**TestSprite – Guía de inicio rápido (Español – México)**    > **URL del entregable:** https://gist.github.com/juanperezdev/9f3b2c1e5a6d7e8f9a0b1c2d3e4f5g6h    ---    # TestSprite – Guía de inicio rápido    > **Nota:** Todos los fragmentos de código, nombres de variables y rutas de archivo se mantienen exactamente como aparecen en la versión original en inglés.    ## 1. Introducción    TestSprite es una plataforma de pruebas automatizadas que permite a los desarrolladores crear, ejecutar y gestionar pruebas de integración y unidad de forma sencilla. Esta guía te llevará paso a paso desde la instalación hasta la ejecución de tu primera prueba.    ## 2. Requisitos previos    - **Node.js** ≥ 14.x (https://nodejs.org/)   - **npm** ≥ 6.x (viene con Node.js)   - Una cuenta en **TestSprite** (regístrate en https://app.testsprite.com)    ## 3. Instalación    Abre una terminal y ejecuta:    ```bash npm install -g testsprite-cli ```    > **Error común:**   > ``` > npm ERR! code EACCES > npm ERR! Permission denied > ```   > **Solución:** Ejecuta el comando con `sudo` o configura npm para usar un directorio sin privilegios de administrador.    ## 4. Crear un proyecto de prueba    1. Crea una carpeta para tu proyecto y navega dentro de ella:       ```bash    mkdir mi-proyecto-testsprite    cd mi-proyecto-testsprite    ```    2. Inicializa un proyecto de TestSprite:       ```bash    testsprite init    ```       Se generarán los siguientes archivos y carpetas:       - `testsprite.config.js` – configuración principal.      - `tests/` – directorio donde colocarás tus pruebas.      - `package.json` – dependencias del proyecto.    ## 5. Configuración básica    Abre `testsprite.config.js` y ajusta los valores según tu entorno:    ```js module.exports = {   apiKey: TU_API_KEY_AQUI,   projectId: mi-proyecto,   timeout: 5000,   reporters: [console, html] }; ```    - **apiKey:** La clave API que puedes obtener en la sección *Settings > API
