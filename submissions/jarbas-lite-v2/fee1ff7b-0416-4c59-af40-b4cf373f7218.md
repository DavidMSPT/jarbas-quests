Introducción  TestSprite es una biblioteca ligera para pruebas de UI que permite crear *sprites* de componentes y compararlos visualmente contra versiones esperadas. Con ella puedes detectar regresiones visuales de forma rápida y fiable.  Esta guía rápida te mostrará cómo instalar TestSprite, crear tu primer sprite y ejecutarlo en tu proyecto.  ---  ### 2. Requisitos previos  - **Node.js** ≥ 14 - **npm** o **yarn** - Un proyecto basado en **React**, **Vue** o **Angular** (cualquiera de los tres está soportado) - Acceso a la terminal de tu sistema operativo  ---  ### 3. Instalación  Abre una terminal en la raíz de tu proyecto y ejecuta:  ```bash npm install --save-dev testsprite ```  o, si prefieres Yarn:  ```bash yarn add --dev testsprite ```  Esto añadirá `testsprite` a tu `devDependencies`.  ---  ### 4. Configuración inicial  Crea un archivo de configuración llamado `testsprite.config.js` en la raíz del proyecto:  ```js module.exports = {   // Ruta donde se guardarán los sprites generados   outputDir: ./testsprite/sprites,    // Ruta donde se almacenarán los snapshots esperados   baselineDir: ./testsprite/baseline,    // Umbral de diferencia aceptable (en %) entre imágenes   tolerance: 0.1,    // Opciones de captura de pantalla   capture: {     viewport: { width: 1280, height: 720 },     deviceScaleFactor: 2,   }, }; ```  > **Importante:** No modifiques los nombres de las propiedades (`outputDir`, `baselineDir`, etc.) ya que TestSprite los lee directamente.  ---  ###
