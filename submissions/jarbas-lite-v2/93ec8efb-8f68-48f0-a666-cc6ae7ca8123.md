Edge‑AI Video Analytics Gateway   **What it is:** A lightweight on‑premise gateway that ingests RTSP streams from nearby cameras, runs TensorFlow‑Lite object detection, and forwards only relevant events to a central data lake.   **Why this spec fits:** The 4 ARM cores (e.g., Ampere A‑64) provide enough parallelism for real‑time inference on low‑resolution feeds, while 24 GB RAM comfortably holds multiple model instances and a small buffer of video frames. The network‑optimized NIC ensures low latency when pushing alerts.   **Deployable tool:** *DeepStream‑Lite* (NVIDIA‑compatible ARM build) or *OpenCV‑DNN* with TensorFlow‑Lite.   **Rough setup:**   1. Spin up an Oracle Cloud Compute instance (ARM shape) and install Docker.   2. Pull the `deepstream-lite:arm64` image, mount camera credentials, and configure model paths.   3. Set up a systemd service to start the container on boot and point its webhook to your central endpoint.    **Estimated monthly usage:** ~2.8 vCPU hrs / day → ~84 vCPU hrs, average RAM 8 GB (≈ 960 GB‑hrs).    ---  ### 2.
