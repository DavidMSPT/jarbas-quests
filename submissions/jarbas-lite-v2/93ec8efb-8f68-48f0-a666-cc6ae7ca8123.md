Edge‑AI Video Analytics Hub   **What it is:** A low‑latency inference node that ingests RTSP streams from on‑prem cameras, runs object detection, and pushes alerts to a central dashboard.   **Why the spec fits:** ARM‑based TensorFlow Lite models run efficiently on 4 cores; 24 GB RAM easily holds multiple concurrent video buffers and the inference graph; the network bandwidth handles several 1080p streams without bottleneck.   **Deployable tool:** **DeepStream‑Lite** (NVIDIA‑compatible ARM build) or **OpenVINO™ Toolkit for ARM**.   **Rough setup:**   - Spin up an Ubuntu 22.04 ARM instance, install Docker.   - Pull the `deepstream-lite:arm64` image and mount camera RTSP URLs.   - Configure the alert webhook to your SIEM and start the container.   **Estimated monthly usage:** ~2.5 vCPU hrs / day (≈75 vCPU hrs), ~12 GB RAM average.    ---  ### 2. Distributed Git Mirror & CI Runner   **What it is:** A private GitHub/GitLab mirror that also hosts lightweight CI jobs for small Go/Node projects.   **Why the spec fits:** 4 ARM cores provide enough parallelism for simultaneous builds; 24 GB RAM prevents OOM on dependency‑heavy npm installs; fast network ensures quick push/pull for remote teams.   **Deployable tool:** **Gitea** + **Drone CI** (both ARM‑compatible).   **Rough setup:**   - Deploy a Docker‑Compose stack with `gitea` and `drone` services.   - Add SSH keys and configure webhooks from upstream repos.   - Create a runner with `drone-runner-docker` pointing to the same instance.   **Estimated monthly usage:** ~1.8 vCPU hrs / day (≈54 vCPU hrs), ~8 GB RAM average.    ---  ### 3.
