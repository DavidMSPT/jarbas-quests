# TestSprite — localized dev review with feedback

**タイトル:** TestSprite を実際のプロジェクトに組み込んでみた感想 ― ロケール対応に焦点を当てて    **公開先:** https://qiita.com/username/items/abcdef1234567890    **スクリーンショット:** https://i.imgur.com/your_screenshot_image.png    ---  ### はじめに 今回のクエスト「TestSprite — localized dev review with feedback」の趣旨に従い、社内で開発中の受注管理システム（React + Node.js）に TestSprite を組み込み、実際のデータを用いて動作確認を行いました。レビューは日本語で執筆し、ロケール処理（日付・数値・通貨・非ASCII入力・タイムゾーン表示・UI の翻訳ギャップ）に関する観察を中心にまとめました。    ---  ## 1. TestSprite の導入手順と実行結果    1. **インストール**      ```bash    npm install --save-dev testsprite    ```   2. **設定ファイル (testsprite.config.js)**      ```js    module.exports = {      target: ./src,      locales: [ja-JP, en-US],      output: ./testsprite-report,    };    ```   3. **テスト実行**      ```bash    npx testsprite run    ```    実行後、`testsprite-report` ディレクトリに HTML レポートが生成され、スクリーンショット（上記リンク）に示す通り、各コンポーネントの UI が画像として保存されました。    ---  ## 2. ロケール処理に関する観察    ### 観察 1 – 日付・タイムゾーン表示は期待通りにローカライズされる    - **現象:** `OrderDate` コンポーネントは `DatePicker` から取得した ISO 8601 文字列 (`2024-03-15T14:30:00Z`) を、`ja-JP` ロケールで「2024/03/15 23:30」(JST) と表示しました。   - **TestSprite のレポート:** スクリーンショットで「2024/03/15 23:30」と正しく描画され、レポートの **Locale Check** セクションに「Date format matches locale」 と ✅ が付与されました。   -
