**TestSprite – Guía de inicio rápido (Español)**    ---    ### 1. Introducción    TestSprite es una herramienta de pruebas de UI basada en sprites que permite crear, ejecutar y mantener pruebas visuales de forma sencilla. Esta guía de inicio rápido le mostrará cómo instalar TestSprite, crear su primer proyecto de pruebas y ejecutar sus pruebas en la línea de comandos.  ---  ### 2. Requisitos previos    - **Node.js** ≥ 14   - **npm** ≥ 6 (viene con Node)   - Navegador Chrome o Chromium instalado   - Acceso a una terminal (PowerShell, bash, etc.)  ---  ### 3. Instalación    ```bash npm install -g testsprite-cli ```  > **Nota:** El comando `-g` instala la herramienta de forma global, lo que permite ejecutar `testsprite` desde cualquier directorio.  ---  ### 4. Crear un proyecto nuevo    1. Abra una terminal y cree una carpeta para su proyecto:    ```bash mkdir mi-proyecto-testsprite cd mi-proyecto-testsprite ```  2. Inicialice el proyecto con TestSprite:    ```bash testsprite init ```  Esto generará la siguiente estructura de archivos:    ``` mi-proyecto-testsprite/ ├─ .testsprite/ │   ├─ config.js │   └─ snapshots/ ├─ tests/ │   └─ example.test.js ├─ package.json └─ README.md ```  - **`.testsprite/config.js`** – Configuración global de TestSprite.   - **`.testsprite/snapshots/`** – Carpeta donde se guardan las capturas de pantalla de referencia.   - **`tests/example.test.js`** – Archivo de ejemplo con su primera prueba.  ---  ### 5. Configuración básica    Edite `.testsprite/config.js` para establecer la URL base de su aplicación:    ```js module.exports = {   baseUrl: http://localhost:3000,   viewport: { width: 1280, height: 720 },   browsers: [chrome] }; ```  - **`baseUrl`** – Dirección donde se ejecuta su aplicación en desarrollo.   - **`viewport`** – Tamaño de la ventana del navegador durante las pruebas.   - **`browsers`** – Navegadores que se utilizarán (solo `chrome` está soportado
